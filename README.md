# beauty_salon




Мой проект «База данных салона красоты» представляет собой учебную реляционную базу данных, спроектированную для моделирования работы современного салона красоты. Проект включает в себя:




Нормализованную схему данных в третьей нормальной форме (3НФ), охватывающую сущности: клиенты, мастера, услуги, записи на приём, косметика, продажи и отзывы.
Полный SQL-скрипт, содержащий:





— создание всех таблиц с внешними ключами,





— наполнение данными (реальные примеры клиентов, мастеров, услуг и транзакций).





Таблицы:





1. clients — Клиенты





Хранит информацию о клиентах салона.





Поля:






client_id — уникальный идентификатор клиента (первичный ключ),

full_name — ФИО,

phone, email — контактные данные,

registracia_data — дата регистрации.

Обеспечивает централизованное хранение данных о клиентах без дублирования.

2. masteri — Мастера

Справочник сотрудников салона (мастеров).




Поля:







rabotnik_id — уникальный ID мастера (PK),

full_name — ФИО,

doljnost — должность (например, «Парикмахер», «Массажист»),

zarplata — оклад,

hire_date — дата приёма на работу.

Вынесено отдельно для соблюдения 3НФ: информация о мастере не дублируется в записях.

3. services — Услуги

Справочник предоставляемых салоном услуг.




Поля:








service_id — уникальный ID услуги (PK),

name — название (например, «Маникюр», «Чистка лица»),

opisanie — описание,

price — стоимость,

duration_seansa — длительность в минутах.

Все данные об услугах хранятся централизованно — изменения применяются в одном месте.

4. zapisi — Записи на приём

Фиксирует факт записи клиента к мастеру на конкретную услугу.




Поля:







zapisi_id — уникальный ID записи (PK),

client_id — ссылка на клиента,

rabotnik_id — ссылка на мастера,

service_id — ссылка на услугу,

vremya — дата и время записи,

status — статус («запланировано», «выполнено», «отменено»).

Центральная связующая таблица между клиентами, мастерами и услугами (связь «многие ко многим» через промежуточную сущность).

5. kosmetika — Косметика

Справочник товаров (косметических средств), продаваемых в салоне.




Поля:






product_id — уникальный ID товара (PK),

name — название,

brand — бренд,

price — цена за единицу,

kolichestvo — остаток на складе.

Данные о товарах вынесены отдельно — соблюдена 3НФ.

6. sales — Продажи

Фиксирует факты продажи косметики клиентам.




Поля:





sales_id — уникальный ID продажи (PK),

client_id — ссылка на клиента,

product_id — ссылка на товар,

vremya — дата и время продажи,

kolichestvo — количество,

summa — общая сумма.

Связывает клиентов и косметику в отношении «многие ко многим» (через промежуточную таблицу).

7. feedback — Отзывы

Хранит отзывы клиентов о посещении салона.




Поля:





otziv_id — уникальный ID отзыва (PK),

client_id — ссылка на клиента,

zapisi_id — ссылка на запись (уникальная),

comment — текст отзыва,

rating — оценка от 1 до 5,

otziv_date — дата отзыва.

Связь с zapisi — «один к одному»: одна запись порождает не более одного отзыва.

Визуализация

Проект сопровождается ER-диаграммой, созданной в draw.io, которая наглядно отображает структуру таблиц, их атрибуты и связи (включая типы: 1:М, 1:1).
